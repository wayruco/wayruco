# License FAQ

## Overview

WayruCO is licensed under the **MIT License**, one of the most permissive open source licenses. This document answers common questions about what this means for you.

## General Questions

### Q: What is the MIT License?

**A:** The MIT License is a permissive open source license that allows you to:
- Use the software for any purpose (commercial or non-commercial)
- Modify the software
- Distribute the software
- Use the software privately

The only requirements are:
- Include the original license and copyright notice
- The software is provided "as is" without warranty

### Q: Is WayruCO really free to use?

**A:** Yes! WayruCO is completely free to use, modify, and distribute. There are no licensing fees or restrictions on commercial use.

### Q: Can I use WayruCO in my commercial product?

**A:** Yes, absolutely. You can use WayruCO in commercial products without paying any fees. You just need to include the MIT License notice in your product.

### Q: Do I have to open source my code if I use WayruCO?

**A:** No. The MIT License is permissive, not copyleft. You can use WayruCO in proprietary software without open sourcing your own code. However, you must include the MIT License notice.

### Q: What if I modify WayruCO?

**A:** You can modify WayruCO for your own use. If you distribute your modifications, you should:
- Include the original MIT License
- Document your changes
- Consider contributing improvements back to the project

## Contribution Questions

### Q: If I contribute code to WayruCO, what happens to my code?

**A:** When you contribute code to WayruCO, you agree that:
- Your code will be licensed under the MIT License
- The WayruCO project can use your code
- Others can use your code under the MIT License terms
- You retain the right to use your code

### Q: Do I need to sign a Contributor License Agreement (CLA)?

**A:** No formal CLA is required. By submitting a pull request, you agree that your contribution can be used under the MIT License.

### Q: Can I use code from other open source projects?

**A:** Yes, but you must:
- Respect the original license
- Include proper attribution
- Ensure license compatibility

For example, you can use MIT-licensed code in WayruCO. GPL-licensed code requires more careful consideration.

### Q: What about the Developer Certificate of Origin (DCO)?

**A:** WayruCO requires a DCO sign-off on commits. This means:
- You certify that you wrote the code or have the right to contribute it
- You understand the code will be used under the MIT License
- You agree to the terms of the DCO

To sign off, add `-s` to your commit:
```bash
git commit -s -m "feat: Add new feature"
```

## Forking & Derivative Works

### Q: Can I fork WayruCO and create my own project?

**A:** Yes! You can fork WayruCO and create your own project. You must:
- Include the original MIT License
- Clearly indicate that it's a derivative work
- Maintain attribution to the original project

### Q: Can I create a proprietary fork?

**A:** You can create a proprietary fork, but you must:
- Include the MIT License for the original code
- Clearly indicate which parts are original vs. modified
- Maintain attribution

### Q: Should I contribute improvements back?

**A:** While not required, we encourage you to contribute improvements back to the main project. This benefits the entire community and helps maintain compatibility.

## Liability & Warranty

### Q: What if WayruCO breaks my system?

**A:** The MIT License includes a disclaimer: the software is provided "as is" without warranty. WayruCO is provided without any guarantee of fitness for a particular purpose.

However:
- We work hard to maintain quality
- We welcome bug reports and fixes
- We provide community support

### Q: Can I sue WayruCO if something goes wrong?

**A:** The MIT License limits liability. You cannot hold the authors or copyright holders liable for damages. However, this doesn't prevent you from:
- Reporting bugs
- Requesting fixes
- Contributing fixes yourself

### Q: What about security vulnerabilities?

**A:** If you discover a security vulnerability:
1. **Don't** post it publicly
2. Email security@wayru.co with details
3. We'll work with you to fix it
4. We'll credit you in the security advisory

## Compliance Questions

### Q: How do I comply with the MIT License?

**A:** To comply with the MIT License:

1. **Include the License**: Copy the MIT License text to your project
2. **Include Copyright Notice**: Include the original copyright notice
3. **Document Changes**: If you modify the code, document your changes
4. **Maintain Attribution**: Keep attribution to the original authors

Example notice:
```
This software includes code from WayruCO (https://github.com/wayruco/wayruco)
Copyright (c) 2025 WayruCO Contributors
Licensed under the MIT License
```

### Q: What if I'm using WayruCO in a closed-source product?

**A:** You can use WayruCO in closed-source products. You must:
- Include the MIT License in your product documentation
- Include the copyright notice
- Make the license available to users

### Q: Do I need to publish my source code?

**A:** No. The MIT License doesn't require you to publish your source code. You can use WayruCO in proprietary software.

### Q: What about patents?

**A:** The MIT License doesn't grant patent rights. If you're concerned about patent issues:
- Consult with a lawyer
- Review the specific patents in question
- Consider the Wayru Network's patent stance

## Trademark Questions

### Q: Can I use the WayruCO name and logo?

**A:** The WayruCO name and logo are trademarks. You can:
- Use them to refer to the original project
- Use them in derivative works (with attribution)
- Use them in documentation and marketing

You cannot:
- Claim to be the official WayruCO project
- Use them in a way that's misleading
- Register them as your own trademark

### Q: Can I create a product called "WayruCO Pro"?

**A:** You should avoid using the WayruCO name in a way that's confusing. Better options:
- "MyCompany's WayruCO Distribution"
- "MyCompany's Internet Access Platform (based on WayruCO)"
- Create your own brand name

If you're unsure, email trademark@wayru.co.

## License Compatibility

### Q: Can I combine WayruCO with other open source projects?

**A:** It depends on the other project's license:

| License | Compatible? | Notes |
|---------|------------|-------|
| MIT | ✅ Yes | Same license |
| Apache 2.0 | ✅ Yes | Both permissive |
| BSD | ✅ Yes | Both permissive |
| GPL v2 | ⚠️ Maybe | Requires careful handling |
| GPL v3 | ⚠️ Maybe | Requires careful handling |
| AGPL | ❌ No | Incompatible |

When in doubt, consult a lawyer or ask the community.

### Q: What if I want to use GPL-licensed code?

**A:** If you use GPL-licensed code with WayruCO:
- Your entire project may need to be GPL-licensed
- You must comply with GPL requirements
- Consult a lawyer for specific situations
